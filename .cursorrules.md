## Project Context

You are working on TrendyReports — a multi-tenant SaaS platform that generates branded real estate market reports from live MLS data.

Stack: Next.js 16 (App Router, Vercel), FastAPI (Render), Celery worker (Render), PostgreSQL + RLS, Redis, Cloudflare R2

We are adding themed PDF generation to all 8 market report types — bringing them to full parity with property reports (5 Jinja2 themes, Smart Color System, server-side PDFShift rendering, R2 upload).

## Rules

1. READ files before modifying them. Always.
2. Follow existing patterns. If similar code exists elsewhere in the repo, match its style exactly.
3. Import, don't copy. If a function exists in another module, import it.
4. No files over 300 lines without good reason.
5. Jinja2 templates: use {% if %} guards for all optional data. Missing field = hidden section, not error.
6. Python: use existing DB patterns (db_conn, set_rls, fetchone_dict). No raw psycopg unless you see it in the file you're modifying.
7. Frontend: Server Components by default. 'use client' only for state/effects/handlers. Tailwind + shadcn/ui only.
8. Always check that existing tests still pass after your changes.
9. When the playbook says "read X file first" — actually read it before writing code.
10. Diagnose before fixing. If something is broken, add logging to confirm the root cause before writing a fix.
