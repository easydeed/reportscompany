# TrendyReports — 8 Market Report Types (V0 Design Spec)

## Overview

Each report is a **single-page, letter-size PDF** (8.5 × 11 in, no scrolling, no page breaks). Every report is agent-branded and themed. All 8 share the same header, footers, and color system — only the **content zone** between header and footer changes.

---

## Shared Anatomy (all 8 reports)

```
┌──────────────────────────────────────────────┐
│  HEADER BAR (gradient)                       │  ← Company logo, company name, report type, agent tagline pill
│  Logo | Company Name / Report Type   [pill]  │
├──────────────────────────────────────────────┤
│  TITLE SECTION                               │  ← "New Listings — Irvine"
│  H1 title + subtitle + accent divider bar    │     "Period: Last 30 days • Source: Live MLS Data"
├──────────────────────────────────────────────┤
│                                              │
│  CONTENT ZONE (varies per layout)            │  ← This is what changes across the 8 types
│                                              │
├──────────────────────────────────────────────┤
│  AGENT FOOTER                                │  ← Agent photo circle, name, title, phone, email, company logo
├──────────────────────────────────────────────┤
│  POWERED-BY FOOTER                           │  ← "Report generated by Company Name • Data source: MLS"
└──────────────────────────────────────────────┘
```

### Color System (CSS Variables)

| Variable | Role | Example (Teal theme) |
|---|---|---|
| `--primary-color` | Headings, prices, dark UI | `#18235c` (user's brand color) |
| `--accent-color` | Dividers, badges, accents | `#0d9488` (theme accent) |
| `--accent-light` | Light tinted backgrounds | `#ccfbf1` |
| `--accent-on-dark` | Accent text on dark bg | `#5eead4` |
| `--accent-on-light` | Accent text on light bg | `#0f766e` |
| `--header-bg` | Header gradient | `linear-gradient(135deg, primary 0%, mid 60%, accent 100%)` |

5 themes available: **Teal**, **Bold**, **Classic**, **Modern**, **Elegant** — each sets different fonts and color roles via the same variable system.

## The 4 Layout Types

### Layout A: Gallery (3 report types use this)

**Visual**: Count badge pill → 3×2 photo card grid (6 cards max)

**Components**:
- **Count badge** — accent-tinted pill: `"117 LISTINGS — LAST 30 DAYS"`
- **Photo card grid** — 3 columns × 2 rows. Each card: property photo (16:10), address, city, price (Georgia serif, large), specs row (bed/bath/sqft icons)
- Grid auto-adjusts: 4 listings → 2×2, 1 listing → single column

### Layout B: Market Narrative (1 report type uses this)

**Visual**: Hero stat pill → AI narrative box → 2×2 compact photo grid → stats bar → quick take

**Components**:
- **Hero stat pill** — same style as count badge: `"$922K MEDIAN SALE PRICE"`
- **AI narrative box** — accent-tinted card with left border: paragraph of market analysis text
- **Compact photo grid** — 2 columns, smaller cards (16:9 aspect), smaller text
- **Stats bar** — horizontal row of 4 stat blocks (Closed Sales, Avg DOM, Months of Inventory, List-to-Sale %) in a light rounded container
- **Quick take** — dark gradient bar with bolt icon + white text summary

### Layout C: Closed/Inventory (2 report types use this)

**Visual**: Hero stat pill → 2×2 photo grid with status badges → data table → stats bar → quick take

**Components**:
- **Hero stat pill** — `"38 TOTAL CLOSED — IN THE LAST 30 DAYS"` or `"67 ACTIVE LISTINGS — CURRENTLY ON MARKET"`
- **Compact photo grid** — 2 columns with status badges overlaid (ACTIVE green, PENDING amber, CLOSED red)
- **Data table** — striped table with columns: Address, Beds, Baths, Sq Ft, Price, DOM. Dark header row. Up to 8 rows.
- **Stats bar** — same as Layout B
- **Quick take** — same as Layout B

### Layout D: Analytics (2 report types use this)

**Visual**: Hero stat pill → stat cards grid → vertical listing rows → quick take

**Components**:
- **Hero stat pill** — `"117 TOTAL LISTINGS ANALYZED — ACROSS MARKET SEGMENTS"`
- **Stat cards** — 4-column grid of bordered cards. Each: large serif value + small label (Median Price, Avg DOM, $/Sq Ft, Close-to-List, Active, New)
- **Listing rows** — vertical list (up to 5). Each row: thumbnail (120×90), address/city/price/specs, status badge pill. Separated by bottom borders.
- **Quick take** — same as Layouts B/C

## Data Available Per Report

Every report receives this data (all fields optional/gracefully degraded):

### Listings (array of up to 6)
```
address, city, list_price, close_price, beds, baths, sqft,
status (Active/Pending/Closed), days_on_market, photo_url
```

### Market Metrics
```
median_list_price, median_close_price, avg_dom,
months_of_inventory, price_per_sqft, list_to_sale_ratio,
new_listings_count
```

### Status Counts
```
Active: 67, Pending: 12, Closed: 38
```

### Branding
```
agent_name, agent_title, agent_phone, agent_email,
agent_photo_url, company_name, logo_url, primary_color
```

### AI Insights
A single paragraph of GPT-generated market commentary (2-4 sentences).

## Design Constraints

1. **Single page only** — everything must fit 8.5 × 11 in with no overflow
2. **Print-ready** — `@page { size: letter; margin: 0; }`, print-color-adjust: exact
3. **No JavaScript** — pure HTML/CSS, rendered to PDF server-side
4. **Font preloading** — Google Fonts loaded via `<link>` in `<head>`
5. **Photo fallback** — SVG building icon placeholder when photo fails/missing
6. **Graceful degradation** — all sections are wrapped in `{% if %}` guards; missing data = section hidden, never broken
7. **Status badges** — color-coded overlays: Active (green), Pending (amber), Closed/Sold (red)
8. **Responsive grid** — cards auto-adjust column count based on listing count (6→3col, 4→2col, 1→1col)
