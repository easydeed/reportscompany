## 1. System Overview

### Two Report Systems (Don't Confuse Them)

**Property Reports** — Single property, 7-page PDF, 5 themes (Bold/Classic/Elegant/Modern/Teal), Jinja2 templates in the worker service. These have their own separate redesign workflow.

**Market/Email Reports** (THIS DOCUMENT) — Multi-listing reports built in `template.py`, sent via Resend email and also rendered to PDF. These are the 8 report types that get **emailed to contacts** via scheduled delivery.

### Target File

`apps/worker/src/worker/email/template.py` — A ~2,020-line Python file that builds email-safe HTML using f-strings. It evolved through 14 versions of incremental improvements. The orchestrator (`send.py`) handles recipient filtering, unsubscribe tokens, and delegates to this template for HTML generation.

---

## 2. Current Architecture

### Tech Stack
- **Not Jinja2** — Pure Python f-string interpolation with inline HTML
- **Email-safe tables** — Proper `<table role="presentation">` layout (not CSS grid/flex)
- **Outlook support** — VML fallbacks for gradient header and circular agent photos
- **Dark mode** — CSS `prefers-color-scheme` support
- **Mobile responsive** — `@media` queries for stacking metrics
- **8 report types** with conditional sections based on `REPORT_CONFIG` flags
- **AI insights** — Calls OpenAI for personalized market commentary, with template fallbacks
- **White-label branding** — Custom colors, logos, agent photos, contact info

## 3. Assessment: What Works vs. What Needs Fixing

### What Works Well
1. **Table-based email architecture** — Proper `<table role="presentation">` everywhere
2. **Outlook VML conditionals** — Gradient header and circular photos have Outlook fallbacks
3. **Dark mode support** — `@media (prefers-color-scheme: dark)` CSS
4. **Mobile stacking** — `@media` queries handle narrow viewports
5. **AI insight integration** — OpenAI-powered market commentary with graceful fallbacks
6. **Report config system** — `REPORT_CONFIG` dict cleanly controls which sections show for each type
7. **White-label branding** — All colors, logos, contact info are parameterized

### The Core Problem

**Brand colors vanish after the header gradient.** Everything below the header is monotone:
- Identical `#fafaf9` background boxes with `#e7e5e4` borders
- Section labels are gray `#57534e` — no brand identity
- No visual hierarchy between the headline metric and supporting data
- Quick Take is invisible — same gray as everything else
- Agent footer feels like an afterthought — white background, plain text links
- Gallery cards are data rows, not marketing pieces

**Result:** The header says "premium real estate brand" but the body says "generic data export."

| CTA button | `bg-[#1e3a5f]` | `background-color: {primary_color}` (already exists) |

### Typography

| Element | V0 Style | Email Translation |
|---------|----------|-------------------|
| Hero metric | `font-serif text-5xl font-bold` | `font-family: Georgia, serif; font-size: 48px; font-weight: 700` |
| Section labels | `text-xs uppercase tracking-widest` | `font-size: 11px; text-transform: uppercase; letter-spacing: 2px` |
| Metric values | `font-serif text-2xl font-bold` | `font-family: Georgia, serif; font-size: 24px; font-weight: 700` |
| Metric labels | `text-[10px] uppercase tracking-wider` | `font-size: 10px; text-transform: uppercase; letter-spacing: 0.5px` |

**Key typography change:** Georgia serif for hero metrics and key numbers — creates the "premium real estate" feel.

## 8. Email Client Compatibility

| Feature | Gmail | Apple Mail | Outlook | Yahoo |
|---------|:-----:|:----------:|:-------:|:-----:|
| `border-radius` | ✅ | ✅ | ❌ (ignored) | ✅ |
| `background-color` hex+alpha | ✅ | ✅ | ❌ | ✅ |
| `linear-gradient` | ✅ | ✅ | ❌ (solid fallback) | ✅ |
| `border-left: 4px solid` | ✅ | ✅ | ✅ | ✅ |
| `font-family: Georgia, serif` | ✅ | ✅ | ✅ | ✅ |
| `<div>` for accent bars | ✅ | ✅ | ⚠️ | ✅ |
| VML `<v:roundrect>` | N/A | N/A | ✅ | N/A |

**Outlook philosophy:** Brand tinting is a progressive enhancement. Outlook users get clean white sections — the layout and typography still work, just without the subtle color washes. This is acceptable.

1. **HTML email table architecture** — All layout = `<table role="presentation">`
2. **VML/MSO Outlook conditionals** — Keep all `<!--[if mso]>` blocks
3. **Dark mode CSS** — Keep `@media (prefers-color-scheme: dark)`
4. **Mobile `@media` queries** — Keep stacking behavior
5. **Helper functions** — `_format_price`, `_get_hero_4_metrics`, etc. are clean
6. **AI insights logic** — Only change the display, not the generation
7. **f-string architecture** — Not switching to Jinja2 for emails
8. **`REPORT_CONFIG` dict** — Section toggle system works well
9. **`send.py` orchestrator** — Don't touch recipient filtering or unsubscribe logic
